Act 2: The Elegant Trick

Hash Map + Linked List = O(1) Everything

The Problem

Operation Hash Map Linked List We Need
Lookup by key O(1) ‚úÖ O(n) ‚ùå O(1)
Track access order No ‚ùå Yes ‚úÖ Yes
Move to front N/A ‚ùå O(1) ‚úÖ O(1)
Evict oldest O(n) ‚ùå O(1) ‚úÖ O(1)

Solution: Use BOTH!

The Architecture

Hash Map: key ‚Üí pointer to list node

Doubly Linked List (access order):
  HEAD ‚Üî [Most Recent] ‚Üî [Key2] ‚Üî [Key3] ‚Üî [Oldest] ‚Üî TAIL

GET key:
  1. Hash map lookup ‚Üí O(1)
  2. Move node to head ‚Üí O(1)

PUT key:
  1. If full, evict tail ‚Üí O(1)
  2. Add to head ‚Üí O(1)
  3. Store in hash map ‚Üí O(1)

LRU Cache Pseudocode

class DListNode:
  key, value
  prev, next // pointers to neighbors

class LRUCache:
  capacity // max items
  cache = {} // HashMap: key ‚Üí DListNode
  head, tail // sentinel nodes (dummy)

GET(key):
  if key not in cache: return -1
  node = cache[key]
  move_to_head(node) // mark as recently used
  return node.value

PUT(key, value):
  if key in cache:
    update node.value
    move_to_head(node)
  else:
    if cache is full:
      evict tail.prev // remove LRU
    create new node
    add to head
    cache[key] = node

The Core LRU Cache

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = {}  # key -> DListNode
        
        # Sentinel nodes (eliminate edge cases!)
        self.head = DListNode()  # Most recent
        self.tail = DListNode()  # Least recent
        self.head.next = self.tail
        self.tail.prev = self.head
    
    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        node = self.cache[key]
        self._move_to_head(node)  # Mark as recently used
        return node.value
    
    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            node = self.cache[key]
            node.value = value
            self._move_to_head(node)
        else:
            if len(self.cache) >= self.capacity:
                lru = self._pop_tail()  # Evict LRU
                del self.cache[lru.key]
            new_node = DListNode(key, value)
            self.cache[key] = new_node
            self._add_node(new_node)

Act 3: What Breaks at Scale

Three ways your LRU cache will crash in production

üî• Disaster #1: Thread Corruption

10 threads, 1 cache, 0 locks

Result: Corrupted linked list ‚Üí SEGFAULT

Thread 1: Removes node B (prev.next = next)
Thread 2: Tries to access node B (already freed!)
‚Üí Undefined behavior, crash, data loss

The Fix: Thread-Safe Wrapper

import threading

class ThreadSafeLRUCache:
    def __init__(self, capacity: int):
        self.lock = threading.RLock()
        self.cache = LRUCache(capacity)
    
    def get(self, key: int) -> int:
        with self.lock:
            return self.cache.get(key)
    
    def put(self, key: int, value: int) -> None:
        with self.lock:
            self.cache.put(key, value)

üî• Disaster #2: OOM Crash

Cache capacity: 1000 items

Each item: 1MB

Result: 1GB memory ‚Üí OOM KILLED

The mistake: Count-based capacity ignores VALUE SIZE!

The Fix: Size-Based Eviction

class SizeAwareLRUCache:
    def __init__(self, max_bytes: int):
        self.max_bytes = max_bytes
        self.current_bytes = 0
        self.cache = {}  # key -> (value, size)
        self.order = DoublyLinkedList()
    
    def put(self, key: str, value: bytes) -> None:
        size = len(value)
        
        # Evict until we have room
        while self.current_bytes + size > self.max_bytes:
            evicted = self.order.pop_tail()
            self.current_bytes -= self.cache[evicted.key][1]
            del self.cache[evicted.key]
        
        self.cache[key] = (value, size)
        self.current_bytes += size
        self.order.add_head(key)

Key insight: Track BYTES, not item count!

class ObservableLRUCache(LRUCache):
    def __init__(self, capacity: int):
        super().__init__(capacity)
        self.hits = 0
        self.misses = 0
        self.evictions = 0
    
    def get(self, key: int) -> int:
        result = super().get(key)
        if result != -1:
            self.hits += 1
        else:
            self.misses += 1
        return result
    
    @property
    def hit_ratio(self) -> float:
        total = self.hits + self.misses
        return self.hits / total if total > 0 else 0.0

Act 4: When LRU Completely Fails

The access pattern that breaks everything

üî• The Sequential Scan Disaster

Database backup: Read all 1M rows

Cache size: 10,000 rows

for row in all_1_million_rows:
  cache.put(row.id, row.data)
  # After 10K rows: first rows EVICTED
  # But next query needs first rows again!

Result: 0% hit rate - Cache is USELESS

Why LRU Fails Here

LRU assumption: "Recently used = likely to be used again"

Sequential scan reality: "Just used = will NEVER need again"

The oldest item (about to be evicted) is actually the NEXT item needed!

LRU Decision Matrix

Access Pattern LRU Works? Better Alternative
Random access ‚úÖ Excellent ‚Äî
Sequential scan ‚ùå 0% hit rate MRU or skip cache
Looping (> cache) ‚ùå Thrashing Larger cache or LFU
Bulk imports ‚ùå Cache pollution Segmented LRU

Know your access patterns before choosing LRU!

Act 5: How the Giants Do It

Redis, Linux, MySQL ‚Äî Same idea, smarter tricks

Redis: Approximate LRU

Problem: Tracking exact LRU for millions of keys = expensive

Solution: Don't be exact ‚Äî be good enough!

def evict_redis_style():
  # Don't scan ALL keys
  sample = random.sample(keys, 5) # Just sample 5!
  oldest = min(sample, key=last_access_time)
  evict(oldest)

# Result: 90% of perfect LRU benefit
# Cost: 10% of perfect LRU overhead

MySQL InnoDB: Segmented LRU

Problem: Full table scans evict hot data

Solution: Split into "young" and "old" segments!

Buffer Pool Split:
  [Young List - 5/8] ‚Üê Frequently accessed data
  [Old List - 3/8] ‚Üê Newly loaded data

New data ‚Üí Old list first
If accessed AGAIN ‚Üí Promote to young list
Full scan ‚Üí Fills old list, young list PROTECTED!

Linux: CLOCK Algorithm

Problem: Full LRU needs timestamp on EVERY memory access

Solution: Hardware provides just 1 bit ‚Äî use it cleverly!

Each page has "referenced" bit (set by hardware)

Clock hand sweeps through pages:
  if referenced_bit == 1:
    referenced_bit = 0 # Second chance!
    move_clock_hand()
  else:
    EVICT this page # Not used since last sweep
